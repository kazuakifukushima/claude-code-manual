13.1 並列 Claude Code 実行で効率を最大化

大規模プロジェクトでは複数の Claude Code インスタンスを同時実行します。

13.1.1 4並列実行の例

# ターミナル1: 認証モジュールのリファクタリング
git worktree add ../proj-auth feature/refactor-auth
cd ../proj-auth && claude "認証モジュールを OAuth2 対応にリファクタリングして"

# ターミナル2: APIエンドポイントの追加
git worktree add ../proj-api feature/new-api
cd ../proj-api && claude "ユーザー管理 API のCRUDエンドポイントを追加して"

# ターミナル3: テスト拡充
git worktree add ../proj-test feature/expand-tests
cd ../proj-test && claude "テストカバレッジを50%から80%に引き上げて"

# ターミナル4: ドキュメント更新
git worktree add ../proj-docs feature/update-docs
cd ../proj-docs && claude "API ドキュメントを最新の実装に合わせて更新して"
Tip並列実行の効果

4つのタスクを並列実行すると理論上4倍速になります。実際には各タスク完了後にマージが必要ですが、全体効率は大幅に向上します（実績: 260%以上の効率化）。

13.2 「スロットマシン」アプローチ

未知の技術スタックや複雑な問題で詰まった場合の戦略です。

状況: 新しいフレームワークへの移行でエラーが続いている

❌ 悪い対処法
- 同じ修正を微調整しながら何度も試す
- Claudeに同じ質問を繰り返す

✅ スロットマシン手法
1. 現在の状態をコミット（「スタート地点」として保存）
2. Claude に「30分で最善を尽くして」と大きな権限を与える
3. 結果を評価する
4. うまくいった → マージして進む
5. うまくいかなかった → git reset でスタート地点に戻り、別のアプローチを試す
# 現状をコミット
git add . && git commit -m "WIP: migration attempt start"

# Claude に大きな権限を与えて実行
claude --dangerously-skip-permissions "Next.js 14 から 15 へのマイグレーションを完了させて。エラーを修正しながら進めてください。"

# 結果を確認
npm run build && npm test

# うまくいかない場合はリセット
git reset --hard HEAD
# 別のアプローチで再試行

13.3 深い思考モードの活用

複雑な問題には、Claude の思考時間を増やすキーワードを使います。

キーワード 効果
think 通常の思考
think hard より深く考える
think harder / ultrathink 最も深い思考（トークン消費多） 
> このシステムのアーキテクチャ設計について ultrathink して、
  スケーラビリティとメンテナンス性の観点から最善の構成を提案して

> このアルゴリズムの時間計算量を think hard して最適化案を示して

13.4 コンテキスト管理のテクニック

長いセッションではコンテキストが不足してきます。

13.4.1 /compact コマンドで圧縮

> /compact
（会話履歴を要約してコンテキストを節約）

13.4.2 重要な情報をファイルに保存する

> この会話で決定したアーキテクチャ方針を `docs/DECISIONS.md` に保存して

> 現在の作業状況を `PROGRESS.md` にまとめて。次のセッションで続きから始められるように。

13.4.3 新セッションで続きを始める

# 前のセッションの記録を参照しながら新セッション開始
claude "PROGRESS.md を読んで、前回の作業の続きから始めて"

13.5.1 段階的実行パターン

> （計画を確認した後）OK、では実装してください

13.5.2 制約明示パターン

> src/auth/ 以外のファイルは変更しないで。
  認証処理のバグを修正して。

13.5.3 ロールプレイパターン

> あなたはシニアセキュリティエンジニアです。

13.6 よくある失敗パターンと対処法

失敗パターン 原因 対処法
指示を無視する CLAUDE.md が長すぎる ルールを優先度順に整理し、最重要のものを先頭に
関係ないファイルを変更する 制約が不明確 「〇〇以外は変更しないこと」と明示
無限ループに入る タスクが曖昧すぎる より具体的な完了条件を定義する
コードが増えすぎる シンプルさの指示がない 「最小限の変更で」と常に指示する
古い方法を使う 最新情報がない バージョン・使用ライブラリを明示する

13.7 セッション間のナレッジ継続

Claude はセッションをまたいで記憶しませんが、ファイルを通じて継続できます。

# .claude/session-context.md（セッション引き継ぎファイル）

## 現在のプロジェクト状況（2026-03-17 更新）

### 完了済み
- ✅ 認証モジュールのリファクタリング
- ✅ ユーザー管理 API の CRUD 実装

### 進行中
- 🔄 テストカバレッジの拡充（現在: 65%、目標: 80%）

### 次のタスク
- フロントエンドのログイン画面実装
- E2E テストの追加

### 重要な決定事項
- JWT の有効期限: 15分（refresh token: 7日）
- パスワードハッシュ: bcrypt（コスト係数: 12）

使い方:

> session-context.md を読んで、現在の状況を把握してから「テストカバレッジの拡充」を続けて
